# Introduction

Welcome to **Collab Digital Twins (CDT)** — a browser-based platform for exploring buildings, sites, point clouds, datasets, and live sensor data on an interactive map. CDT bridges BIM and GIS using open standards, so you can work with multi-scale geospatial information without leaving the browser.

This section is for **people using the CDT platform** — to view a building, browse open data for a city, monitor sensors, or collaborate around a site.

## What you can do

- **Explore on a map** — pan, zoom, and layer geospatial data using the [Map Viewer](./guides/map-viewer.md)
- **View 3D buildings and BIM models** with the [BIM Viewer](./guides/bim-viewer.md)
- **Inspect point cloud scans** of real-world environments via the [Point Cloud Viewer](./guides/point-cloud-viewer.md)
- **Browse buildings and sites** and their associated documents, plans, and metadata ([guide](./guides/buildings-and-sites.md))
- **Discover open datasets** from connected open data portals ([guide](./guides/datasets-and-open-data.md))
- **Monitor live sensors and IoT devices** mapped to your assets ([guide](./guides/sensors-and-iot.md))
- **Manage files** attached to buildings, sites, and datasets ([guide](./guides/file-management.md))
- **Collaborate** by sharing comments and annotations on assets ([guide](./guides/collaboration.md))

## Where to start

- **First time here?** Run through the [Quickstart](./getting-started/quickstart.md) for a 10-minute tour of the interface.
- **Setting up your environment?** Follow the [Installation guide](./getting-started/installation.md).
- **Want background context?** The [Concepts](./concepts/digital-twins.md) section explains digital twins, BIM, GIS, point clouds, and the open standards CDT builds on.

## Core values

CDT is built around a few principles that shape how the platform behaves:

- **Open standards** — interoperability over proprietary lock-in
- **Open source** — transparent, community-driven development
- **Browser-native** — no specialized software to install
- **Multidisciplinary** — designed for stakeholders across BIM, GIS, planning, and operations
